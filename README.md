# MMFC 评分校准展示版

本项目用来验证“评分者标准不一致”会怎样影响作品排名，以及校准算法能不能降低这种运气因素。

它不会上传数据，也不会改网站代码。所有计算都在本机浏览器里完成。

展示版内置数据已经脱敏：网站用户 ID、报名显示名、评论行 ID、作品提交行 ID 和报名行 ID 都替换为匿名编号，评论里的外部图片和链接也会被隐藏。页面不提供参数调节控件，使用固定展示口径计算。

## 这个工具想解决什么问题

当前活动里作品很多，但每个参与者通常只会评价其中一小部分作品。这样会产生两个问题：

- 有些作品刚好被偏严格的人评分，均分会偏低。
- 有些作品刚好被偏宽松的人评分，均分会偏高。

这个实验台会尝试估计每个评分者的“给分习惯”，再把他们的评分换算到更接近全体共识的尺度上。

简单说：

```text
原始分数
  -> 判断评分者是偏严、偏松，还是分差尺度不同
  -> 把每条评分校准到统一尺度
  -> 重新计算作品分数和排名
  -> 对比校准前后差异
```

## 启动

```powershell
py -m http.server 8765 --bind 127.0.0.1
```

然后打开：

```text
http://127.0.0.1:8765/
```

页面默认会加载 `data` 目录下的三张表：

- `BOFcomment.csv`
- `作品提交.csv`
- `比赛报名.csv`

也可以在页面上手动导入新的评论表、作品表、报名表 CSV；导入数据会先在浏览器内按同一规则脱敏，再进入计算和导出。

## 展示版固定计算口径怎么理解

`校准模型`

- `仿射校准 x' = a x + b`：同时修正评分者的整体偏高/偏低，以及分差尺度。
- `只校准偏移 x' = x + b`：只修正整体偏高/偏低，不改变评分者拉开分差的幅度。

建议先看“只校准偏移”，再看“仿射校准”。仿射模型更强，但也更容易受样本结构影响。

`作品风险统计最少评分数`

这个数值不再决定是否写排名。所有有有效评分的作品都会排名；这个阈值只用于风险提示和“排名大幅变化”等汇总统计，避免少量评分作品把风险列表挤满。

`评分者参与校准最少评分数`

评分次数太少的人，不足以判断他是偏严还是偏松。少于这个数量的评分者会默认不校准。

`参数收缩基准 N`

用来防止过拟合。评分次数越少，校准参数越接近“不校准”。

直观理解：

```text
评分 3 次的人：只轻微校准
评分 30 次的人：可以更相信他的偏差估计
```

`异常偏移阈值`

用于标记风险评分者。例如阈值是 80，表示某个评分者在 600 分附近需要修正超过 80 分，就会被标记为“偏差较大”。

## 校准流程

### 1. 清洗数据

先把 CSV 中不能参与正式评分的记录排除掉：

```text
全部评论
  -> 排除楼中楼回复
  -> 排除作者自评
  -> 排除无效分
  -> 得到正式评分样本
```

每条正式评分会变成这样的结构：

```text
评分者 j 对作品 n 给了 x_nj 分
```

其中：

- `n` 是作品。
- `j` 是评分者。
- `x_nj` 是原始分数。

### 2. 先算一个原始作品分

一开始还不知道谁偏严、谁偏松，所以先用当前网站口径算作品均分。

如果开启 Qualified 权重，则：

```text
作品原始分 =
(Q评分总和 * 2 + 普通评分总和) / (Q评分数 * 2 + 普通评分数)
```

这个初始作品分记作 `q_n`。

### 3. 估计每个评分者的给分习惯

对每个评分者，工具会查看他评过的作品。

如果某个评分者给作品的分数经常低于这些作品的当前均分，说明他可能偏严。

如果经常高于当前均分，说明他可能偏松。

仿射模型写成公式就是：

```text
q_n ≈ a_j * x_nj + b_j
```

含义是：

- `x_nj`：评分者原本给出的分。
- `q_n`：当前估计的作品分。
- `a_j`：这个评分者的分差尺度。
- `b_j`：这个评分者的整体偏移。

然后把他的原始分转换为校准分：

```text
x'_nj = a_j * x_nj + b_j
```

如果只使用偏移模型，就相当于固定：

```text
a_j = 1
```

也就是只做加减分修正。

### 4. 用校准后的分数重新算作品分

当每个评分者都有了校准参数后，工具会把所有评分换成校准分，再重新计算每个作品的分数。

```text
所有 x_nj
  -> 变成所有 x'_nj
  -> 重新计算每个作品 q_n
```

如果开启 Qualified 权重，重算时仍然按 Q 用户更高权重计算。

### 5. 重复直到稳定

第 3 步和第 4 步会反复执行：

```text
估计评分者参数
  -> 重算作品分
  -> 再估计评分者参数
  -> 再重算作品分
  -> 直到变化很小
```

页面里的“迭代收敛”图就是在看每一轮作品分数最大变化量。越快接近 0，说明算法越稳定。

## 为什么要做参数收缩

如果一个人只评了很少作品，直接估计他的偏严/偏松会很危险。

例如某人只评了 3 个作品，而且刚好这 3 个作品都比较特殊，那么算法可能误判他整体偏严或偏松。

所以工具会做参数收缩：

```text
lambda = count / (count + shrinkN)
```

其中 `count` 是评分次数。

评分次数越少，`lambda` 越小，校准越弱。

最终参数是：

```text
a = 1 + lambda * (a_fit - 1)
b = lambda * b_fit
```

直观理解：

```text
评分很少：更接近原始分
评分很多：更相信校准结果
```

## 页面结果怎么看

### 总览指标

这里先看数据是否足够健康：

- 有效正式评分：进入计算的评分数量。
- 已写排名：所有有有效评分、能计算均分的作品数量。
- 评分矩阵密度：评分覆盖率，越低说明大家评价重叠越少。
- 高风险评分者：偏差超过阈值的人数。
- 排名大幅变化：校准前后排名变化很大的作品数量。

### 分数分布

看整体评分是否集中在合理区间。

如果高分或低分异常集中，可能说明评分参考标准本身需要调整。

### 作品评分数覆盖

看每个作品有多少评分。

评分数越少，单个评分者对作品结果的影响越大。

### 评分者活跃度

看每个评分者评了多少作品。

评分次数越多，这个人的校准参数越可信。

### 原始分 vs 校准分

每个点是一个作品。

- 接近对角线：校准前后变化小。
- 明显偏离对角线：这个作品受评分者偏差影响较大。

### 排名变化最大作品

显示校准后排名变化最大的作品。

这些作品不一定有问题，但值得人工看评分明细。

### 评分者校准参数

横轴是评分次数，纵轴是这个评分者在 600 分附近的修正幅度。

- 正数：算法认为他偏严，所以会抬高他的分。
- 负数：算法认为他偏松，所以会压低他的分。
- 离 0 越远：评分尺度和群体平均差异越大。

### 风险提示

这里会自动列出需要重点检查的评分者或作品，例如：

- 某个评分者修正幅度过大。
- 某个评分者斜率异常。
- 某个作品校准后排名变化很大。
- 某个作品评分分歧很大。

### 评分者诊断表

重点看这些列：

- 次数：评分样本数。
- 均值偏移：相对其他评分者共识的偏移。
- `a`：分差尺度。
- `b`：整体偏移。
- 600分修正：最直观的偏严/偏松指标。
- 校准轮数：该评分者实际参与参数拟合的全局迭代轮数；未参与校准时为 0。
- 状态：是否稳定，或是否偏差较大。

在“评分者诊断”里点击评分者 ID，会在新窗口打开这个人的独立诊断页；详情页内再点击作品或评分者，也会继续用新窗口打开对应对象。

## 推荐查看顺序

建议按这个顺序看结果：

1. 使用固定展示口径，先看总览指标和风险提示。
2. 查看“评分者诊断”，确认是否确实存在明显偏严/偏松的人。
3. 查看“排名变化最大作品”，逐个点进单作品明细。
4. 导出作品结果和评分者结果 CSV，做人工复核。

如果某些作品在当前展示口径下变化很大，说明这些作品确实受评分者抽样影响较大，适合优先人工复核。
