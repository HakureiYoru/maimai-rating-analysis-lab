# MMFC 评分校准实验台

在浏览器本地验证：评分者标准不一致对排名的影响，以及校准能否减弱这种「运气」因素。

## 算法概要

**问题**：每人只评部分作品，偏严/偏松的评分者会使作品均分系统性偏高或偏低。

**思路**：估计每位评分者的给分习惯，把每条评分换算到更接近全体共识的尺度，再重算作品分与排名。

```text
原始分数 → 估计评分者偏差 → 校准每条评分 → 重算作品分/排名 → 对比前后
```

### 1. 数据与初始作品分

正式样本排除楼中楼、作者自评、无效分。每条记录为：评分者 `j` 对作品 `n` 给出 `x_nj`。

先用当前口径（可选 Qualified 权重 ×2）算作品均分 `q_n`，作为后续拟合的参照。

### 2. 评分者模型

对评分者 `j`，用其评过的作品拟合：

```text
q_n ≈ a_j · x_nj + b_j
```

- `a_j`：分差尺度（仿射模型）
- `b_j`：整体偏移

校准分：`x'_nj = a_j · x_nj + b_j`。若只用**偏移模型**，则固定 `a_j = 1`，只做加减修正。

### 3. 迭代

交替执行：用当前 `q_n` 更新各评分者 `(a_j, b_j)` → 用校准分重算各作品 `q_n`，直至变化足够小。

### 4. 参数收缩

评分次数少时，直接估计偏差易过拟合。设 `λ = count / (count + shrinkN)`：

```text
a = 1 + λ · (a_fit - 1)
b = λ · b_fit
```

评分越少，`λ` 越小，结果越接近原始分。
